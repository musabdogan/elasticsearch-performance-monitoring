Privacy

Last updated: March 2025

Usage of personal data

This extension does not collect or use any personal data at all.

Usage of your Elasticsearch data

This extension does not analyze the data in your Elasticsearch cluster.

This extension does not send data from your Elasticsearch cluster to any other service.

Monitoring data (cluster health, node stats, index stats, etc.) is fetched only for display in the dashboard. It is not stored permanently or transmitted elsewhere.

Local storage

The Extension stores the following on your device only, using Chrome's storage API:

  • Cluster configurations: URLs and optional credentials (username/password or API key) for clusters you add manually.
  • Preferences: Theme (dark/light), polling interval, alert settings, selected cluster.

Your cluster configurations and credentials remain on your device and are never sent to us or any third party.

Browser extension permissions

The Chrome extension requires the following permission:

  • Read and change all your data on the website you visit (Chrome) / Access your data for all websites (Firefox)

This is the same permission name for different browsers. In theory, the extension could read and change data on any website you visit. This is not what the extension intends to do.

This extension does not read or change any data of any websites that you visit, and it will never do that.

The only reason for this permission is convenience: you can connect to your Elasticsearch cluster without configuring CORS. You can start using the extension right away without any cluster configuration. This is common practice for browser addons that communicate directly with Elasticsearch.

Third-party services

The Extension does not use any third-party analytics, advertising, or data collection services. All functionality is self-contained.
